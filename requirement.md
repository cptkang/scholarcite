
# Product Requirement Document (PRD): ScholarCite AI (Platinum v7.5.0)

## 1. 프로젝트 개요
- **프로젝트 명:** ScholarCite AI
- **목적:** 연구자의 초안 문구 및 데이터를 분석하여 실존하는 학술 논문을 검색하고, 학계 무결성을 보존하며 인용구를 삽입하는 지능형 에디터.
- **핵심 가치:** 100% 실존 근거 기반 인용(No-Hallucination), 연구 세션별 근거 인벤토리 관리, 사용자 선택형 인용 제어 시스템.

## 2. 주요 기능 요구사항

### 2.1 문장 및 데이터 분석 (Input)
- **멀티 포맷 지원:** 일반 텍스트 및 HTML 표(Table) 데이터를 인식하여 분석.
- **PDF 분석:** PDF 파일을 업로드하여 단락별로 텍스트를 추출하고 분석 대상 문장을 선택할 수 있는 인터페이스 제공.

### 2.2 인용 생성 및 교정 (Core AI Engine)
- **근거 우선 인용 (Evidence-Only Citation):** Google Search Grounding을 통해 실존함이 확인되고, 논문 내 구체적인 원문 문장(Original Sentence)이 매칭되는 경우에만 인용을 생성함.
- **인용 생략 정책:** 적절한 학술적 근거가 발견되지 않을 경우 할루시네이션 방지를 위해 해당 부분의 인용 삽입을 생략함.
- **재구성 모드:**
    - **Refine Mode:** 학술적 문체로 문장을 다듬으면서 인용 삽입.
    - **Keep Mode:** 원본 문장을 100% 보존하면서 인용만 삽입.

### 2.3 학술 증거 인벤토리 (Evidence Inventory)
- **세션 기반 관리:** "분석 인용 생성"을 수행한 시점(Execution Session)별로 결과를 그룹화하여 관리.
- **원문 증거 노출 (Verified Evidence):** 인용된 논문의 제목, 저자, 저널 정보와 함께 **논문 내 실제 원문 문장**, **전체 단락 문맥**, **원문 위치(Section)**를 명확히 표시.
- **선별적 인용 제어 (Selective Citation Control):**
    - 각 근거 항목 옆에 체크박스를 배치.
    - 체크박스 선택 해제 시, "분석 결과" 텍스트에서 해당 인용 태그(예: [1], (Lee, 2024))가 실시간으로 제거됨.

### 2.4 비교 및 검증 (Comparison)
- **Diff 시각화:** 원본 문구와 인용이 적용된 교정 문구 사이의 변경 사항을 단어 단위로 비교하여 강조 표시.
- **무결성 레포트:** AI가 해당 인용을 선택한 학술적 사유(Citation Reason)를 제공하여 연구자의 판단을 도움.

### 2.5 내보내기 및 관리 (Export)
- **참고문헌 포맷:** EndNote 전용 Import 포맷(.enw) 지원.
- **증거 보고서:** 선택된 세션 혹은 전체 세션의 근거 자료를 Markdown(.md) 또는 MS Word(.doc) 파일로 내보내기 지원.
- **개인 키 지원:** 사용자 본인의 Gemini API 키를 등록하여 개별 할당량 사용 가능.

## 3. UI/UX 디자인 명세
- **테마:** 고대비 Slate & Indigo 학술 테마.
- **레이아웃:** 
    - 좌측: 네비게이션 및 세션 히스토리.
    - 중앙/우측: 분석 도구 및 결과 뷰어.
- **반응성:** 인용 선택 여부에 따른 본문 텍스트의 실시간 반응형 업데이트(Reactive Update) 보장.

## 4. 기술 사양
- **모델:** `gemini-3-flash-preview` 최적화 사용.
- **도구:** `googleSearch` Grounding 필수 활용.
- **무결성 검증:** `originalSourceSentence` 및 `citationTag` 필드를 통한 엄격한 데이터 매칭 시스템.
