# Product Requirement Document (PRD): ScholarCite AI

## 1. 프로젝트 개요
- **프로젝트 명:** ScholarCite AI (Platinum v6.9.2)
- **목적:** 연구자의 초안 문구 및 데이터를 분석하여 실존하는 학술 논문을 검색하고, 학술적 문체로 재구성하며 인용구를 삽입하는 지능형 에디터.
- **주요 가치:** 논문 작성 시간 단축, 인용의 정확성 확보, 데이터 기반 문장 생성.

## 2. 핵심 기능 요구사항

### 2.1 문장 및 데이터 분석 (Input)
- **리치 텍스트 지원:** 사용자는 일반 텍스트뿐만 아니라 웹, 엑셀, 워드에서 복사한 **표(Table)**를 직접 붙여넣을 수 있어야 함.
- **UI 구조:** 입력창은 `contentEditable`을 사용하여 서식을 유지하며, 내용이 길어질 경우 내부 스크롤이 발생하고 하단의 '분석 실행' 버튼은 항상 고정되어 노출되어야 함.

### 2.2 학술적 재구성 및 인용 (AI Engine)
- **한국어 최적화:** 모든 분석 결과(Cited Text)는 논문 투고 수준의 전문적인 한국어로 작성됨.
- **실존 논문 검색:** Google Search Grounding을 활용하여 실존하는 논문의 제목, 저자, 저널, URL을 추출함.
- **표 출력 지원:** 입력 데이터가 수치 비교를 포함하거나 표 형식일 경우, 결과물 역시 HTML Table 태그(`<table>`, `<tr>`, `<td>`)를 활용하여 구조화된 표로 출력할 수 있어야 함.
- **인용 스타일 적용:** 사용자가 선택한 스타일(APA, IEEE, Vancouver, Chicago, MLA)에 맞춰 본문 내 인용 표기 삽입.
- **저널 등급 필터링:** SCI, KCI, Scopus, JCR Q1/Q2 등 목표 등급에 부합하는 논문을 우선적으로 검색.

### 2.3 문서 관리 및 내보내기 (Output)
- **비교 분석 모드:** 원본 문구와 AI가 수정한 문구를 단어 단위로 비교하여 변경 사항을 시각화(Diff)함.
- **참고문헌 관리:** 검색된 논문의 상세 정보(id, 저자, 연도, 저널, 인용 사유)를 카드 형태로 제공.
- **EndNote 연동:** 분석된 모든 참고문헌을 EndNote Import Format(.enw) 파일로 다운로드 가능.

## 3. 기술 사양 (Technical Specs)

### 3.1 AI 및 데이터
- **모델:** `gemini-3-flash-preview` (속도 및 할당량 최적화).
- **검색 도구:** `googleSearch` 도구를 필수 사용하며, `groundingMetadata`를 통해 출처 URL 검증.
- **API Key 관리:** `process.env.API_KEY`를 사용하며, 사용자가 직접 자신의 유료 API 키를 등록할 수 있는 `window.aistudio.openSelectKey()` 인터페이스 제공.

### 3.2 UI/UX 디자인
- **레이아웃:** 사이드바(설정) + 메인(워크스페이스)의 2컬럼 구조.
- **스크롤 정책:** 입력 영역과 출력 영역은 독립적인 스크롤을 가지며, 화면 전체 레이아웃이 깨지지 않도록 `overflow-hidden` 처리.
- **컴포넌트:** Tailwind CSS를 활용한 고대비 학술적 디자인 (Indigo & Slate 테마).

## 4. 사용자 인터페이스(UI) 주요 요구사항
- **반응형 대응:** 문장이 아무리 길어져도 실행 버튼이 화면 아래로 밀려나지 않도록 카드 내부 스택 레이아웃 적용.
- **에디터 편의성:** 분석 결과물 섹션 내에서 사용자가 직접 문구를 수정할 수 있는 편집 기능 제공.
- **에러 핸들링:** API 할당량 초과(429) 시 사용자에게 친절한 안내와 함께 API 키 등록 버튼 제공.

## 5. 향후 확장성
- PDF 파일 업로드 및 단락 단위 일괄 인용 생성 기능 (기능 구현됨).
- 구글 독스(Google Docs)와의 실시간 동기화 및 문서 생성 기능 (Status 정의됨).
